# Temporal Graph Open Follow Ups

Several follow-ups could improve the temporal graph controls after more hands-on testing.

- Confirm whether five minutes is the right grouping threshold for [[Smart Time Bucket Filtering]].
- Add an explicit reset-to-`All time` affordance if users need a faster path back from a narrow span.
- Consider visualizing the active span on the track so users can see which portion of history is included.
- Test whether graph auto-follow is too eager for large graphs and whether it needs damping or a settling threshold.
- Decide whether test timeline nodes should remain in the repository or be replaced by fixtures.

These questions relate to [[Time Filter UI Contract]], [[Graph Camera Follow Behavior]], and [[Timeline Test Nodes]].
